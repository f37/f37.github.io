Sequence Prediction via Mixture Density Network (MDN)¶

Ich möchte gerne die Struktur meiner Arbeit vorstellen und im Anschluss eine grobe Übsersicht über die Resultate des Programmierprojektes geben.

Abstract¶

Als großer Fan von Predictive Processing möchte ich ein MDN(Bishop 1994) ins leben rufen. Das MDN ist ein Modul, welches den Output eines Künstlichen Neuronalen Netzes stochastisch (bayesian framework) interpretiert. Somit kann jede Form von Netzwerkarchitektur zur Crossvalidierung verwendet werden (MLP, LSTM, ResNet,...).

Das MDN ist ein generatives Modell, welches die zugrundeliegende (bedingte) Dichtefunktion der Daten approximiert. Hiermit kann man Sensor-/Motorsignale, bzw. jede Form von sequentiellen Daten vorhersagen und gleichzeitig mit dem MDN ein Maß für die Sicherheit der Vorhersage angeben.

Note:¶

Tensorflow bietet auch eine direkte Anbindung an die GPU. Aufgrund der Größe der Daten (audiodateien ~ 44,1kHz) müsste ohne dieses feature das "training" des Netzes in Wochen gezählt werden.

Daten¶

Im Moment habe ich das Modell auf Toydaten, Tonsignalen (kontinuierlich) und MIDI (discret) trainiert. In der Inferenz wird der Output autoregressiv als Input verwendet. Der Algorithmus ist nicht auf nicht beschränkt auf den Input. Jeder Sequentielle Datensatz (diskret oder kontinuierlich) verwendet werden, e.g. Sensor/Motor Signale. Das Modell approximiert die zugrundeliegende (bedingte) Dichtefunktion der Daten.

Hyperparameter/Architektur¶

Zum Abschluss würde ich die verschiedenen Architekturen miteinander crossvalidieren. Wie im Abstract erwähnt ist das MDN ein Modul, was auf den Outputunits eines Neuronalen Netzes gebaut wird. Also kann man sich in der Wahl des Netzes frei entscheiden, und muss schauen welche Architekturen bzw. Hyperparameter am Besten zum Problem passen.
